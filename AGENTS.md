# Project Working Rules

This repository is a Next.js praise community app. Any coding agent that opens this project must treat repository state, PR state, and verification as part of the task.

## Start Here

Before changing files, run and read:

```bash
git status --short --branch
git worktree list
git branch --show-current
git remote -v
```

If the task is continuing pull request work, also run:

```bash
gh pr status
gh pr checks --watch=false
```

If GitHub access fails because of network sandboxing, request approval and retry with `gh`. Do not assume PR state from memory.

## Branch and PR Flow

- Use `codex/` branch names for new work unless the user says otherwise.
- Continue an existing PR branch when the user asks to continue existing work.
- After PR merge, update local `main` from `origin/main` before starting the next branch.
- Keep unrelated dirty files out of commits. If a dirty file predates the task, name it in the final response and leave it unstaged.
- Use `gh` to inspect PRs and checks when working on GitHub-backed changes.

## Verification

Run verification before claiming work is complete or pushing PR changes:

```bash
npm run verify
```

`npm run verify` intentionally runs lint, tests, production build, and TypeScript checking in sequence. Do not run `npm run build` and `npx tsc --noEmit` in parallel because `.next/types` may be in flux while the build is running.

For Prisma schema changes, also run:

```bash
npm run prisma:generate
npx prisma migrate status
```

For local database-backed behavior, make sure migrations have been applied with the appropriate Prisma command before testing the app manually.

## Product Context

- The product is a Korean-first praise community. Use "칭찬" in Korean-facing copy.
- Users write praise request posts. AI writes praise comments, not posts.
- AI praise is generated by the BullMQ worker in `src/server/jobs.ts`.
- Gemini is the default AI provider; OpenAI is switchable through environment configuration.
- AI usage controls are database-backed and managed through `/moderation`.
